RAG (Retrieval-Augmented Generation) — это метод улучшения генерации текста.
Основная идея: перед генерацией ответа модель получает релевантный контекст из базы знаний.
Это снижает количество галлюцинаций (hallucinations) и повышает фактическую точность.
RAG особенно полезен для вопросно-ответных систем и диалоговых ассистентов.
Процесс RAG состоит из нескольких этапов: индексация документов, поиск, генерация.

На этапе индексации документы разбиваются на чанки (куски текста).
Каждый чанк превращается в вектор с помощью модели эмбеддингов.
Векторы сохраняются в векторной базе данных для быстрого поиска.
FAISS — библиотека от Meta для эффективного поиска по векторам.
Она поддерживает различные типы индексов для баланса скорости и точности.
IndexFlatIP — точное косинусное сходство без потери качества.

На этапе поиска вопрос пользователя тоже превращается в вектор.
Вектор вопроса сравнивается со всеми векторами в базе данных.
Находятся K ближайших чанков по косинусному сходству.
На этапе генерации найденные чанки добавляются в промпт модели.
Модель формулирует ответ на основе предоставленного контекста.
Такой подход позволяет модели отвечать на вопросы по документам.
